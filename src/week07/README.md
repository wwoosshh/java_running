# Week 7 — 미니 은행 시스템 (Mini Bank System)

> 콘솔 기반 다중 계좌 은행 프로그램. Week 6 ATM의 다음 단계.

---

## 📌 프로젝트 개요

한 명의 사용자가 자기 계좌 1개만 다루던 ATM(Week 6)에서 한 단계 진화.
**여러 사용자, 여러 계좌, 그리고 계좌 간 송금**까지 다루는 작은 은행 시스템을 만듭니다.

사용자는 계좌를 개설하고, 입출금하고, 다른 계좌로 송금하고, 잔액과 거래 내역을 조회할 수 있습니다.

---

## 🎯 학습 목표

이 프로젝트를 끝내면 다음을 할 수 있게 됩니다.

- [ ] 한 클래스가 **여러 객체를 소유하고 관리**하는 구조 설계 (composition)
- [ ] 컬렉션에서 **식별자(ID)로 객체 찾기** 패턴 구현
- [ ] 객체를 못 찾은 상황을 `null` 반환으로 알리고, 호출자가 안전하게 처리하기
- [ ] **두 객체에 걸친 작업** (송금 = 한쪽 출금 + 한쪽 입금) 구현
- [ ] handler 측 검증 + Account 측 검증의 **이중 방어선** 새 도메인에 적용
- [ ] try-catch 패턴을 한 메뉴 시스템 안에서 일관되게 사용

## 📝 메뉴별 동작 명세

### 1. 계좌 개설
```
예금주 이름: 김지훈
1번 계좌 개설 완료. (예금주: 김지훈, 잔액: 0원)
```

### 2. 입금
```
계좌번호: 1
입금할 금액: 10000
1번 계좌에 10000원 입금 완료. 현재 잔액: 10000원
```

계좌가 없으면:
```
계좌번호: 99
계좌번호 99번을 찾을 수 없습니다.
```

### 3. 출금
입금과 동일한 흐름. 잔액 부족 시 `"잔액 부족. 현재 잔액: X원"`.

### 4. 송금
```
출발 계좌번호: 1
도착 계좌번호: 2
송금할 금액: 3000
송금 완료. 1번 → 2번, 3000원
```

실패 케이스:
- 출발 계좌 없음 → `"출발 계좌 99번을 찾을 수 없습니다."`
- 도착 계좌 없음 → `"도착 계좌 99번을 찾을 수 없습니다."`
- 잔액 부족 → `"송금 실패: 잔액 부족."`
- 음수 송금 → `"송금 실패: 금액은 양수여야 합니다."`
- 출발 == 도착 → `"송금 실패: 출발 계좌와 도착 계좌가 같습니다."`

### 5. 잔액 / 거래 내역 조회
```
계좌번호: 1
[1] 김지훈 — 잔액: 7000원
===== 거래 내역 =====
+10000
-3000
```

### 6. 전체 계좌 목록
```
[1] 김지훈 — 잔액: 7000원
[2] 이서연 — 잔액: 3000원
```

비어있으면: `"개설된 계좌가 없습니다."`

### `Bank.transfer` 의사코드 (이걸 먼저 종이에 적어보세요)

```
1. fromNumber로 출발 계좌 찾기 (findByNumber)
   → null이면 false 반환
2. toNumber로 도착 계좌 찾기
   → null이면 false 반환
3. from == to 면 false 반환 (자기 자신에게 송금 금지)
4. from.withdraw(amount) 시도
   → 실패면(false) 그대로 false 반환
5. to.deposit(amount) 호출
6. true 반환
```

> 4번에서 출금이 실패하면 잔액은 그대로 (withdraw가 알아서 거부함).
> 5번에서 입금이 실패하면? — 출금은 이미 됐는데 입금이 안 됐다면 돈이 사라짐!
>   현재 설계에서 deposit이 실패하는 유일한 경우는 amount가 음수일 때.
>   withdraw가 먼저 음수를 거부하므로 5번이 실패할 일은 없습니다.
>   (불안하면 도전 B 참고)

## 📌 의도적으로 빼둔 것 (난이도 폭증 방지)

- **HashMap** — List 순회로 충분한 규모. Map은 더 나중에.
- **Stream / Lambda** — for-each + if 만으로 모든 검색 가능.
- **상속 / 인터페이스** — 계좌 유형(보통예금/적금)은 도입하지 않음.
- **사용자 정의 예외 클래스** — boolean 반환과 null 반환으로 충분.
- **파일 저장 / DB** — 인메모리. 종료하면 다 사라지는 게 정상.
- **인증 / 비밀번호** — 계좌번호만으로 접근.

이 항목들은 Week 8 이후 또는 Spring 단계에서 본격적으로 만납니다.
